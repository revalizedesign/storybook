## Design review — first pass (Matt)

This first group is a strong start, especially because it establishes canonical names and maps several patterns back to ShadCN primitives. That alone helps create shared language.

That said, most of these are still incomplete at the story level. Per-component notes live in each story's Docs tab.

### Common gaps

- Deprecated states and variants.
- Real product examples.
- Mobile and limited-space examples.
- Localization stress testing.
- Edge cases.
- "Do this / not this" guidance.
- More complete rules and usage rationale.
- Clearer distinction between primitive, component, pattern, and product-specific behavior.

### Roadmap focus

- Increase coverage across the priority patterns.
- Add real product examples wherever possible.
- Use deprecated variants to clarify what we are moving away from.
- Stress test patterns against real constraints.
- Divide ownership so we can improve coverage quickly.
- Prioritize patterns that are most likely to create inconsistency at the page or product level.
